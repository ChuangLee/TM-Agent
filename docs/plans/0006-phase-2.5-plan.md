> **状态(2026-04-21):已完成 + 存档。** PR1–6 全部落地 (`5c3b94f` → `03e4818`);`action_first` flag 已拆除,见 [ADR-0006 addendum](../adr/0006-mobile-action-first-ui.zh.md#addendum-2026-04-21-拆除-action_first-debug-flag)。本文保留作为历史执行记录——文中凡提到 `useActionFirstFlag()` / `?action_first=1` 的段落**已过时**,以当前代码 + ADR addendum 为准。

## 总体目标

用"shell 状态驱动的动作面板 + 顶部下拉按键层"替换移动端"compose bar 主输入"模型。桌面端不动。整个 Phase 2.5 在 `action_first` 调试 flag 之后,Phase 1 UI 作为回退保留到 flag 默认翻开。

## 时间估算

总计约 **10–15 个工作日**,跨 5 个 PR。单 PR 设计目标:每个可独立 merge + 部署到 `your-host.example` + flag-gated 不影响默认用户。

| PR                             | 估时   | 重点                                                                     |
| ------------------------------ | ------ | ------------------------------------------------------------------------ |
| PR1 分类器 + 后端信号          | 2 天   | 后端协议扩展 + 纯函数分类器 + hook                                       |
| PR2 ActionPanel 骨架 + 2 状态  | 2 天   | 卡片带组件 + `shell_idle` + `editor`                                     |
| PR3 剩余 6 状态卡片            | 3 天   | `tui` / `repl` / `pager` / `confirm` / `password` / `long_process`       |
| PR4 KeyOverlay 顶部下拉层      | 3 天   | 半透明覆盖层 + 粘滞修饰键 + 下拉手势                                     |
| PR5 Compose 扩展               | 2–3 天 | per-session 历史、草稿、快速插入、剪贴板粘贴                             |
| PR6 桌面直通模式 (Direct Mode) | 3–4 天 | `keydown` → PTY 映射、IME 托管、非 Surface 区域模糊 + 呼吸光晕、多路退出 |

风险缓冲已经包含在每 PR 估时内。

## 依赖与顺序

```
PR1 ──┬─► PR2 ───► PR3 ───┬─► PR5
       │                     │
       ├────► PR4 ───────────┘
       │
       └────► PR6 (桌面直通,可独立并行)
```

- PR2 和 PR4 都只依赖 PR1(分类器 + 信号);可以并行推,但约定先 merge PR2 以尽早验证 `action_first` flag 的集成路径。
- PR3 扩展 PR2 的面板,线性依赖。
- PR5 可在 PR3 / PR4 任一 merge 后启动,逻辑独立。

## 通用跨 PR 约束

- **Debug flag**:所有新 UI 必须经过 `useActionFirstFlag()` 门控。flag 来源:`localStorage.tm-agent_action_first === '1'`,或 URL `?action_first=1`,或 build-time env(开发环境默认开)。关闭 flag 时前端行为与当前 Phase 1 完全一致。
- **部署验证**:每个 PR 合入后必须 `sudo -n systemctl restart tm-agent.service` + 真机过手机一次(主 + flag 开关路径)。
- **Playwright 基线**:每个 PR 交付时更新 `tests/frontend/e2e/*` 保证 flag=off 的所有现有用例通过,并为新功能加对应用例。
- **CI 必须绿**:typecheck + unit + integration + e2e + lint 全过才可 merge。不得 `--no-verify` 跳钩子。

## PR1 — 分类器 + 后端信号

**分支名**:`feat/action-first-classifier`

**目标**:把"shell 状态"做成可被 UI 订阅的一等状态。后端推送 `pane_current_command`,前端纯函数分类,Zustand slice 暴露。

**交付物**:

- 后端
  - `src/backend/tmux/tmux-gateway.ts`:扩展 `listPanes` 的 format string,追加 `#{pane_current_command}`。
  - `src/shared/protocol.ts`:`paneSummary` Zod schema 增加 `paneCurrentCommand: z.string()` 字段。
  - 现有 snapshot 推送路径无需改,只是字段多了。
- 前端
  - `src/frontend/features/shell-state/classifier.ts`:纯函数 `classify(signals): { state: ShellState; confidence }`,按[调研报告](../research/0006-mobile-action-first-research.md) §2.2 的修正后正则。
  - `src/frontend/features/shell-state/state-definitions.ts`:`ShellState` 枚举、每状态的元数据占位(卡片列表为空,PR2/PR3 填充)。
  - `src/frontend/features/shell-state/use-shell-state.ts`:订阅 xterm `onWriteParsed`(200ms debounce)+ `onBufferChange` + Zustand snapshot 的 `paneCurrentCommand`,产出 `ShellState`。
- 测试
  - 单元:`classifier.test.ts` 表驱动,至少复制调研 §2.3 的 18 条 + 加 12 条覆盖 PowerShell / nushell / fish / git-pager 过渡等 §2.4 列出的空白。
  - Integration:假 xterm buffer + 假 tmux snapshot 注入 → 断言 `useShellState()` 输出正确且 debounce 生效。
- 文档
  - `docs/design/0006-mobile-action-first-tech-design.md` 第 1 章 "状态机" 落地到代码后回填。

**验收标准**:

1. `classify()` 30+ 条 fixture 全绿。
2. Zod schema 兼容:前端收老格式(无 `paneCurrentCommand`)时不抛错(字段 `.optional()` 或给默认空串,任选其一并 ADR 记录)。
3. `flag=off`:`useShellState()` 仍被调用但输出被 UI 忽略(无视觉变化);现有 Playwright 全部通过。
4. `flag=on`:浏览器 DevTools 能从 Zustand 看到 `shellState` 实时变化——手测切 vim / 切 bash / 运行 `sudo` 出现 password 提示,三种状态肉眼能分辨。

**回滚**:单 commit revert 即回到旧 snapshot 协议;前端 hook 删掉无副作用。

## PR2 — ActionPanel 骨架 + `shell_idle` + `editor`

**分支名**:`feat/action-panel-shell`

**目标**:落地卡片带组件,先把 2 种最常见状态的卡片接上,跑通"flag 开 → 看到卡片 → 点卡片 → PTY 反馈"的完整回路。

**交付物**:

- `src/frontend/features/action-panel/ActionPanel.tsx`:**响应式** 横向卡片带——`< 820px`(移动)两行 96px + 44px 卡,`≥ 820px`(桌面)单行 40px + 28px 卡,CSS 切换。移动端固定在 `Surface` 和 `ComposeBar` 之间,桌面端固定在 `TopBar` 和 `Surface` 之间,对应 `App.tsx` grid 多出一行。
- `src/frontend/features/action-panel/Card.tsx`:单张卡片。props `{label, payload, kind}`;单击 → 发送 payload,长按(500ms) → 填入 compose bar 供编辑。
- `src/frontend/features/action-panel/use-desktop-shortcuts.ts`:桌面键盘 `Alt+1` … `Alt+9` 绑定到前 9 张卡片。
- `src/frontend/features/action-panel/desktop-shortcut-registry.ts`:模块级 card registry,供 Direct Mode 的 `keydown-to-bytes` 询问"这个 Alt+数字是否被 ActionPanel 用掉了"(PR6 要用)。
- `src/frontend/features/action-panel/state-definitions.ts`(PR1 占位的扩展):
  - `shell_idle`:`ls` / `cd ..` / `git status` / `git diff` / `npm run dev` / `clear` / `↑ history` / `Tab` / `⌨` / `✎`。
  - `editor`:`:w` / `:q` / `:wq` / `i` / `Esc` / `/` / `gg` / `G` / 方向键四向 / `⌨` / `✎`。`nano` 检测时提供 `^O` / `^X` / `^K` 替代。
- `useActionFirstFlag()` hook + flag 门控 `App.tsx`(flag off 时 `ActionPanel` 不渲染)。
- Playwright:开 flag → shell 空闲 → 点 `ls` → 验证 terminal 出现 `ls` 输出;进入 vim → 验证 `:w` 卡片出现。

**验收标准**:

1. `flag=on` 手机打开 your-host.example,肉眼看到 96px 卡片带贴在终端下方;空闲态显示 idle 卡,进 vim 显示 editor 卡。
2. `flag=on` PC(≥ 820px)TopBar 下方出现 40px 紧凑卡片带;按 `Alt+1` 发送第 1 张卡片的 payload。
3. 点 `ls` 在终端看到正确输出;长按 `ls` 发现 compose bar 被预填 `ls`。
4. `flag=off` UI 完全等于 merge 前,新旧 Playwright 基线都绿。
5. 卡片带不影响 Surface 的高度测量(feedback_grid_sizing_race 的红线):`tmux list-clients` 显示正常 cols×rows,不缩回 12 行;桌面 40px 额外行也不破坏 1fr。

**回滚**:`action-panel` 目录整删 + `App.tsx` 恢复旧 grid。

## PR3 — 剩余 6 状态卡片 + prompt 捕获横幅

**分支名**:`feat/action-panel-all-states`

**目标**:铺完 8 种状态的卡片集,并把 `confirm_prompt` / `password_prompt` 做成 banner 级 UI(不仅是卡片)。

**交付物**:

- 每状态的卡片集(见[产品设计](../design/0006-mobile-action-first-product-spec.md,PR3 前必须 locked):
  - `tui`:`y` / `n` / `Esc` / `/` / `?` / `Enter` / `Ctrl+C` / 方向键 / `⌨` / `✎`
  - `repl`:`↑ history` / `Tab` / `.exit` / `.help` / 常用关键字(按语言变体)/ `⌨` / `✎`
  - `pager`:`Space` / `b` / `/` / `n` / `q` / `G` / `gg` / `?` / 方向键 / `⌨` / `✎`
  - `confirm_prompt`:两个大按钮 Yes/No,检测到的默认选项视觉加重
  - `password_prompt`:原生 `<input type="password">` + 隐/显切换 + 发送/取消
  - `long_process`:单个大 `Ctrl+C` 按钮 + 实时监听的"按 h 显示帮助"式卡片提取
- `src/frontend/features/action-panel/PromptCaptureBanner.tsx`:`confirm_prompt` / `password_prompt` 下把卡片带换成 banner(全宽、加重、不可横滑),密码字段绝不进 compose 历史或 draft。
- Playwright:对 8 种状态各写一条 happy-path。
- 集成测试:调研 §2.5 提到的 `git` → `less` → `git` 过渡抖动,断言 200ms debounce 生效,卡片带不抖。

**验收标准**:

1. 8 状态 × 1 Playwright 用例全绿。
2. 在真机上触发 `sudo apt install foo`,看到 Yes/No banner,点 Yes 后流程继续。
3. 在真机上触发 `sudo ls`,看到密码 banner;输入的字节不出现在 DevTools Zustand 的 `compose.history` 或 `compose.draft`。
4. Git 展开 pager 时卡片平滑切换,无闪烁。

**回滚**:单独 revert 此 PR 不影响 PR2;回到 2 状态可用。

## PR4 — KeyOverlay 顶部下拉按键层

**分支名**:`feat/key-overlay`

**目标**:用户点 `⌨` 或从屏幕顶部下拉,弹出半透明覆盖层,容纳低频键 + 修饰键 + F-keys。与 compose 互斥。

**交付物**:

- `src/frontend/features/key-overlay/KeyOverlay.tsx`:top-drop 层,~68vh,半透明背景,用 CSS `transform: translateY()` + `@keyframes` 做 180ms 入场。
- `src/frontend/features/key-overlay/key-layout.ts`:按 `ShellState` 返回上部"状态相关键"列表,其余区块通用。
- **粘滞修饰键**(借鉴自 tmux-mobile,见[调研报告](../research/0006-mobile-action-first-research.md) §1.3):单点 toggle → 长按 lock → 再点 release。视觉琥珀色边框表示 armed,红色表示 locked。
- **F1–F12 行**:默认折叠,点 `Fn` 展开。
- `src/frontend/hooks/use-top-edge-pull.ts`:从屏幕顶部边缘下拉 60px 触发 overlay 呼出。
- 手势互斥:overlay open 时点 `✎` 自动关 overlay + 呼出 compose;反之打开 compose 强制关 overlay。
- Playwright:呼出 overlay → 点 `Ctrl` + `C` → 验证 PTY 收到 `\x03`;点 `F2` 展开 Fn 区 → 点 `F1`。

**验收标准**:

1. 真机验证:iPhone Safari / Android Chrome 各一次,overlay 入场流畅(≥ 55fps),半透明背后终端仍可辨识。
2. 粘滞修饰键:点 `Ctrl` 琥珀高亮 → 点 `C` 发送 `\x03` 并 Ctrl 自动释放;长按 `Ctrl` → lock 红高亮,连续多次发送 Ctrl-X 直到再次长按解除。
3. Overlay 关闭方式三种都能用:点 bottom 32vh、上滑、点 `✎`。
4. `prefers-reduced-motion` 下无 transform 动画,只 opacity。

**回滚**:`key-overlay` 目录整删 + `ActionPanel` 的 `⌨` 卡片改为 no-op 占位(或也一并去掉)。

## PR5 — Compose bar 扩展

**分支名**:`feat/compose-bar-extensions`

**目标**:把 compose bar 从"光秃秃的输入框"升级成小型产品,集中处理历史、草稿、快速插入、粘贴。

**交付物**:

- `src/frontend/features/compose/history-store.ts`:Zustand slice,按 session id 存 `{text, state, ts}[]`。首次 `select_session` 时机会性拉取 `~/.bash_history`(通过 backend /api/history endpoint,后端读文件并返回,加大小限制)。
- 上箭头 / swipe-up 在空 compose 上触发历史回溯;按当前 `ShellState` 过滤(vim 态只显示 `:` 前缀条目)。
- `src/frontend/features/compose/draft-store.ts`:per-session 草稿暂存;切 session 保留,刷新丢失(v1 内存)。
- 快速插入(长按 compose bar):从最近 N 行 buffer 正则抽取 {URL、绝对路径、git 分支名},弹出小托盘选择后插入。
- 剪贴板粘贴按钮(借鉴自 tmux-mobile):compose bar 右侧图标按钮,`navigator.clipboard.readText()` → 插入光标位置。权限被拒时 toast 提示手动粘贴。
- 后端可能需要:`GET /api/history?session=<id>` 读 `~/.bash_history` 或 `HISTFILE`(如果暴露),有大小限制(512KB)和防路径遍历。
- 单元 + 集成:历史过滤、草稿持久性、剪贴板错误回退。

**验收标准**:

1. 真机:切 session 两次,每次草稿不丢。
2. vim 状态下按 ↑,只看到 `:w`、`:q`、`:e file` 这种 `:` 前缀的历史。
3. 长按 compose → 看到 URL/path/branch 托盘,点一下插入成功。
4. 剪贴板按钮:iPhone Safari 首次触发权限弹窗,允许后正常工作;拒绝后 toast 提示且不崩。
5. 密码态下:发送后草稿立即清空,历史里没有密码条目(回归 PR3 验收 #3)。

**回滚**:回退 `compose-bar-extensions` 分支;旧 compose bar 继续工作。

## PR6 — 桌面直通模式 (Direct Mode)

**分支名**:`feat/desktop-direct-mode`

**目标**:PC 用户一键切入"直接键盘驱动"模式,像传统 webshell 一样用。视觉强引导,避免用户不知道自己在这个状态。

**交付物**:

- `src/frontend/features/direct-mode/DirectModeProvider.tsx`:Context + 状态机(`idle` / `entering` / `active` / `exiting`),包住 `App.tsx`。
- `src/frontend/features/direct-mode/use-direct-mode.ts`:hook,提供 `{ active, enter, exit, toggle }`;内部管理 IME textarea focus、keydown capture、paste capture。
- `src/frontend/features/direct-mode/DirectModeIndicator.tsx`:顶部 40px accent 色悬浮条,左侧脉冲点 + "直通中 · `Ctrl+]` 退出" 文案。
- `src/frontend/features/direct-mode/keydown-to-bytes.ts`:键盘事件 → xterm/ANSI 字节串映射。至少覆盖:字母/数字/符号、方向键、Home/End/PgUp/PgDn、F1-F12、Tab、Enter、Backspace、Delete,以及 Ctrl/Alt/Shift/Meta 组合。**Alt+1-9 特例放行**:若 `desktop-shortcut-registry` 有对应 slot 则返回 `null`(交给 ActionPanel 处理),否则按常规 Alt+数字发送 PTY。
- `src/frontend/features/direct-mode/ime-bridge.tsx`:隐藏 `<textarea>`,监听 `compositionstart/update/end`,合成期 UI 显示小浮层,合成结束整段发送 PTY。
- `src/frontend/features/shell/TopBar.tsx` 改造:`md` 断点 + `pointer: fine` 时显示直通按钮(一键切换)。
- CSS:`body[data-direct-mode="active"]` 选择器下,非 `.tm-scroller` 祖先的 UI 元素加 blur + 过渡;Surface 四周 `box-shadow` breathing 动画。
- 单元 + 集成:
  - `keydown-to-bytes`:表驱动,覆盖常见键 + Ctrl/Alt 组合。
  - `use-direct-mode`:模拟按键事件 → 断言 PTY 收到正确字节;`Ctrl+]` 不发 PTY 而触发 exit。
  - IME 模拟:模拟 compositionstart/end,断言合成期无转发、合成结束整段发送。
- E2E(Playwright,desktop viewport 1280×800):
  - 点直通按钮 → indicator 出现 → 按 `ls\n` → terminal 有输出 → 按 `Ctrl+]` → 退出、indicator 消失、非 Surface 区恢复清晰。
  - 进入态下试按 `Cmd+W`(mock browser不会关):验证事件不影响 shell,也不 crash。
  - 中文 IME 路径模拟:触发 composition 序列 → 断言 PTY 只收到合成完成的 payload。

**验收标准**:

1. PC (1280×800) 打开 your-host.example,TopBar 右上有"直通模式"按钮;点它进入。
2. 进入后:非 Surface 区块肉眼可辨地变模糊、灰度、透明;顶部悬浮条清晰(accent 色)带脉冲点;Surface 周围有呼吸光晕。
3. 进入后按普通命令 `ls<Enter>`、vim 进出、htop 滚动、`cat foo.txt` 全部如桌面 ssh 一样自然。
4. `Ctrl+]` 或 "退出直通" 按钮或 Esc 双击(300ms 窗口)三种方式都能退;退出后视觉完全复原。
5. 中文 IME(搜狗/微软)下输入中文 → 合成时不发 PTY,合成完成发完整字符串(不乱码,不吞字)。
6. 手机(390×844)打开站点时**不显示**直通按钮(`pointer: coarse` 过滤);`?direct_mode=1` URL 参数仍能手动开(允许用户绕开)。

**回滚**:revert `feat/desktop-direct-mode` 分支;桌面回到现有 compose bar + sidebar 模式,不受其他 PR 影响。

**风险**:

- R6.1 浏览器保留键不可抓(`Cmd+W` 等):在帮助提示里明示。
- R6.2 IME 在 macOS Safari vs Chrome 差异:PR6 合入前双端过一次真机。
- R6.3 `Ctrl+]` 被某些终端程序用(emacs `C-]`):indicator 同时提示备选 "Esc Esc";文档里写明"若程序需要 `Ctrl+]`,请用 Esc 双击退出"。

## 里程碑外的事项

- **文档**:每个 PR merge 时同步更新
  - `docs/design/0006-mobile-action-first-tech-design.md`(代码级差异)
  - 各 PR 的 PR 描述 / changelog
  - 截图贴到 PR 描述(特别是 UI PR)
- **性能**:`useShellState()` 的 rAF + debounce 在每 PR 集成时记录 profile 数据(帧时间、CPU),在 PR5 之后出一份汇总。
- **无障碍**:PR4 需测 VoiceOver / TalkBack 能读出 overlay 的 role 和 按键标签。Phase 6 会再做系统性无障碍 pass。
- **i18n**:v1 只做中英文双语常量(中文为 fallback,英文源字符串)。详见技术设计。
- **Flag 切换策略**:Phase 2.5 全部 merge 之后,留 1-2 周双轨(flag 默认关),期间内部真机用 flag=on 完整跑过所有 8 状态 ≥ 3 次,再 PR 把 flag 默认翻开(并在下一个 release cycle 里彻底删除 Phase 1 fallback 代码)。
