## 风险与回滚

- **风险**:catalog 里某条 slash 命令在某 CLI 新版被改名 → 用户选了无效命令。
  **对策**:用户能立即看出来(命令报错),改 catalog 一行重发。无系统性影响。
- **风险**:iOS 软键盘失焦闪烁。
  **对策**:tech-design §6.4 已规定 `pointerdown.preventDefault`;PR1 部署后立即在真机验。若无法靠 preventDefault 解决,降级方案 = `keepFocusOnPick=true` flag 再深挖。
- **回滚**:任何 PR 都可独立 revert。整个 feature 不在 critical path 上(不影响发送、不影响 classifier、不影响 Direct Mode)。
